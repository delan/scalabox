How does it work? return

Scalabox was created because manually calculating the values necessary for a truly scalable interface is tricky. Making this worse, two separate cases are needed; one for when the viewport is wider than the box's aspect ratio, and one for when it is narrower.

When the viewport is too wide, all values are in terms of vh, and the height is (100 − 2 × margin) vh. When it's too narrow, everything's in vw, and the width is the same number of vw units.

In addition to both of the cases being automatically generated, scalabox also generates fallback code for browsers that don't support the vw and vh units, providing for a fixed-size box that's still centered and still has the correct font size.

When building interfaces with a scalable box, using absolute positioning for all elements and percentages for all dimensions is optimal. In the preview, the green square is 1em × 1em, and the button is 25% × 15%.

One downside to using this technique is that you can't set relative font sizes with em or % because they won't scale; instead, classes for each 'relative' font size must be defined with pre-calculated size values in terms of vw and vh. Thankfully, this is also done for you for several basic sizes.

About scalabox return

Scalabox allows you to easily generate CSS for truly scalable (but fixed, not responsive) interfaces positioned centrally on screen, with a defined aspect ratio, margin and font size relative to the box height. It uses the new CSS vw and vh units and aspect-ratio media queries.

As of August 2012, the styles generated by scalabox are supported fully in Chrome and Internet Explorer, falling back to a non-scalable box in Firefox, iOS, Android and other browsers.

Scalabox is written by Delan Azabani. It's released under the MIT license and can be found on GitHub.
